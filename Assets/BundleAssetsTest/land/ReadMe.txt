---------------------------------------------------文件夹描述-----------------------------------------------------------------------------------------
Tile 放置地表的基础地块和地表的一些修饰预制(包含_TYPE_是基础地块60*60规格)
Tile_lod0 存放原始格式的（静态树和动态树还没合批的）
Tile_lod1 存放大地图使用的格式3*3瓦片+山脉+河流+湖泊+峡谷+动态树(已合批)+静态树(已合批)
Tile_lod2 存放游戏内使用的格式，




--------------------------------------------------万国地表描述----------------------------------------------------------------------------------------
材质：
	瓦片材质lod0:   4层纹理+Mask融合  第四层的纹理可以通过全局的 场景颜色进行变更。  256*256 + 1024*1024
	瓦片材质lod1:   4层颜色+Mask融合  第四层的纹理可以通过全局的 场景颜色进行变更。
	瓦片材质lod3:   4层颜色+Mask融合  第四层的纹理可以通过全局的 场景颜色进行变更。

瓦片: Mask 复用 只是四层纹理搭配不一致而已
 1： 草土  4个方向
 2： 草雪  4个方向
 3:   土草  4个方向
 4:   草土  4个方向
 5:   雪草  4个方向
 6:   纯土 +  1点点草      mask
 7:   纯沙漠
 8:   纯草 +  1点点土      mask
 9:   透明地块

lod 0 Tile  显示详细的 3*3瓦片 采用lod0材质 
lod 1 Tile  显示详细的 3*3瓦片 采用lod1材质    +   物件原始名称
lod 2  Tile  显示详细的 3*3瓦片 采用lod1材质   +   物件原始名称_LOD1
lod 3  Tile只有一块 采用材质lod2 +  物件原始名称_LOD2  移除树木
lod 4  Tile只有一块 采用材质lod3 +  物件原始名称_LOD3  移除树木
lod 5  Tile只有一块 采用材质lod5 +  模型combinemesh合并

场景配置文件:
tile_data: 基础大瓦片的信息   （大瓦片地表 大瓦片建筑） （大瓦片装饰物树林） （大瓦片名称） 目前大地图是有 137 种模板
map_data: 地表数据  地图 40*40的瓦片 id 号 旋转，河流方向，  省份信息（2*2）桥是否可用
场景显示规则:

lod 0 1: 加载   Tile 信息(瓦片和瓦片内的建筑)  和 装饰物（主要是树林)       
lod 2 3 4 5 :  清楚掉 lod01的 瓦片和树林  数据，  加载 对应lod的 瓦片预制体  Tile_xxxxxxx_lod2

-------------------------------------------------------------自动LOD-------------------------------------------------------------------------
修饰物 rok 为 树，我们目前不只有树

Lod0: 9个基础地表  +  地形模型(山脉，峡谷，河流，山丘)  +  动态修饰物(Grove)  +  静态修饰物(9个区域的需要合并)
Lod1: 复用lod0
Lod2: 9个基础地表  + 地形模型lod1  + 动态修饰物中间的那个Sprite + 静态修饰物(分9个区域合并)(合并的内部自己分组合并)（并且合并完数量要减半）
lod3: 1 个 地表  + 地形模型 lod 2  + 动态修饰物中间的那个Sprite（减半） 
lod4: 1 个 地表  + 地形模型 lod 2  + 动态修饰物中间的那个Sprite（减半） 
lod5: 1 个地表 + 山合并模型 + 其它合并模型

其中，河流的只有 lod0 1 是预制体模型。  其它的都是变成一张图片（所以美术对于地图上的河流还需要有图片的）
